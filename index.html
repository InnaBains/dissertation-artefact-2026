Project overview

Integrating nutrition, sustainability and pantry reuse

This artefact addresses the gap identified in the dissertation: most existing applications treat nutrition tracking, environmental sustainability and household food waste reduction separately. The system combines nutritional composition data, CO₂e indicators and pantry‑aware logic into a single, interpretable decision‑support framework for UK university students.

Nutritionally balanced weekly plans
Recipes are evaluated against NHS Eatwell dietary guidelines using the UK CoFID 2021 dataset, supporting balanced weekly meal planning rather than isolated single‑meal choices.
CO₂e‑aware sustainability indicators
Environmental impact data from Poore & Nemecek (2018) is mapped to recipe ingredients, enabling relative CO₂e labels and sustainability categories for each meal.
Pantry‑aware, waste‑risk‑sensitive logic
The system models a student pantry, prioritising ingredients close to expiry, maximising reuse of existing items and dynamically deducting quantities when recipes are marked as cooked.

Key system features

Designed around student realities and explainable decisions

The system’s rule‑based engine and interface are informed by literature on student food behaviours, household food waste and decision‑support explainability, focusing on transparency rather than black‑box optimisation.

Pantry‑aware recommendation behaviour
Recipes are scored against current pantry contents, prioritising high‑match recipes that reuse ingredients already available in the student’s kitchen.
Expiry‑date prioritisation
Ingredients are classified by freshness and waste‑risk, enabling recommendation modes that bring forward items approaching their use‑by date.
Sustainability‑aware meal planning
CO₂e‑based labels and sustainability categories help students compare the relative environmental impact of different weekly meal plan options.
Explainable recommendation logic
Rule‑based scoring and SQL analytical views expose how each recipe was selected, making the system auditable for academic evaluation and understandable for end users.
Dynamic pantry deduction
When a recipe is marked as cooked, ingredient quantities are deducted from the pantry, closing the loop between planning and actual usage.
Integrated nutrition + sustainability + pantry framework
Unlike many existing tools that treat these areas separately, this artefact unifies nutritional optimisation, environmental impact and food‑waste prevention in one system.

Architecture & data pipeline

From raw datasets to explainable recommendations

The system integrates three core datasets (CoFID 2021, Poore & Nemecek (2018), and a Kaggle recipe dataset) through a Python preprocessing pipeline, storing analytical outputs in a MySQL database accessed via a Spring Boot API and presented through a React.js frontend.
