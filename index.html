Project Overview

Integrating nutrition, sustainability, and pantry utilisation

This artefact responds to a key gap identified in the dissertation: existing tools typically address nutrition tracking, environmental sustainability, and household food waste in isolation. The system integrates nutritional composition data, CO₂e indicators, and pantry‑aware logic into a single, interpretable decision‑support framework tailored for UK university students.

Nutritionally balanced weekly plans
Recipes are assessed using NHS Eatwell guidance and the UK CoFID 2021 dataset, supporting balanced weekly planning rather than isolated meal choices.
CO₂e‑aware sustainability indicators
Ingredient‑level environmental impact data from Poore & Nemecek (2018) enables relative CO₂e scoring and sustainability categorisation for each recipe.
Pantry‑aware, waste‑risk‑sensitive logic
The system models a student pantry, prioritising items nearing expiry, maximising ingredient reuse, and deducting quantities dynamically when meals are marked as cooked.
Technology Stack

Full‑stack implementation with transparent logic

The artefact is implemented as a modern web application comprising a React.js frontend, a Spring Boot backend, a MySQL analytical database, and a Python preprocessing pipeline for integrating nutrition, sustainability, and recipe datasets.

Key System Features

Designed for transparency and student‑centred decision‑making

The system’s rule‑based engine is informed by literature on student food behaviours, household food waste, and explainable decision‑support. The focus is on clarity and interpretability rather than black‑box optimisation.

Pantry‑aware recommendation behaviour
Recipes are scored against current pantry contents, prioritising high‑match options that maximise ingredient reuse.
Expiry‑date prioritisation
Ingredients are categorised by freshness and waste‑risk, enabling recommendations that bring forward items approaching expiry.
Sustainability‑aware meal planning
CO₂e‑based labels and sustainability categories help students compare the environmental impact of weekly meal plan options.
Explainable recommendation logic
Rule‑based scoring and SQL analytical views make the system’s decision process transparent and academically auditable.
Dynamic pantry deduction
When a recipe is marked as cooked, ingredient quantities are deducted automatically, linking planning with real‑world usage.
Integrated nutrition–sustainability–pantry framework
The system unifies nutritional assessment, environmental impact, and food‑waste prevention within a single coherent framework.

Architecture & Data Pipeline

From raw datasets to transparent recommendations

The system integrates three core datasets—CoFID 2021, Poore & Nemecek (2018), and a Kaggle recipe dataset—via a Python preprocessing pipeline. Processed outputs are stored in a MySQL analytical database, accessed through a Spring Boot API and presented through a React.js frontend.
